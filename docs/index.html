一、系统概览

本系统是一个云端自动化市场数据报告工具，运行在 GitHub Actions 上。每个交易日自动生成多份 HTML 报告，推送到 GitHub Pages 供手机/电脑随时查看。所有数据来自公开 API（东方财富、新浪、中行），不涉及交易执行，仅供数据参考。

二、报告清单与时间表

报告时间（北京时间）内容
🏙️ 早盘报告08:15美股隔夜收盘、亚太早盘、商品期货、全球要闻、今日预判
🌤️ 午盘分析11:35市场广度（涨跌比）、行业板块排行、概念资金流、持仓快照
🌙 收盘总结15:05六大指数收盘、行业板块、概念资金、期货、外汇、要闻
🧠 量化仪表盘按需触发20层量化指标：市场状态、BSI板块强度、资金流、主力行为、成本区、突破、Meta评分
📊 周报分析周五 15:45本周指数走势图、行业资金汇总、期货周变化、深度文字复盘
📝 生成日志自动追加每次报告生成的时间、状态（成功/部分异常/失败）和错误详情

三、持仓管理

首页「📌 我的持仓」区域支持：

• 添加股票：点击 + 添加，输入6位代码和名称，可选填成本和备注

• 编辑/删除：点击已有股票即可编辑，点击 ✕ 删除

• 重点关注：点击 ⭐ 标记，标的会置顶并高亮显示

• 数据持久：所有修改自动保存到浏览器本地存储，不会丢失

四、管理面板

首页下方齿轮 ⚙️ 连点 3次 打开管理面板，可以手动触发云端重新生成各类报告。点击按钮会跳转到 GitHub Actions 页面，需要手动点击 Run workflow。

⚠️ 手动触发需要 GitHub 仓库写权限。报告生成约需 40-90 秒，完成后刷新对应页面查看。

五、数据新鲜度说明

• 每份报告页脚显示 北京时间 YYYY年MM月DD日 HH:MM，为实际生成时刻

• 午盘报告含新鲜度检测：正常时段采集完整数据；延迟到13:00后标注"数据可能含下午盘"；延迟到15:00后跳过行业/概念资金流（避免收盘数据误导）

• 生成日志页 可查看所有报告的状态和历史错误信息

• 量化仪表盘标注数据源：REAL（实时）/CACHE（缓存）/FALLBACK（降级）

六、手机端使用

• 在手机浏览器打开 GitHub Pages 地址，添加到主屏幕（iOS Safari 分享 → 添加到主屏幕）

• 已适配移动端布局，报告卡片、表格均可正常浏览

• GitHub Actions 查看执行历史和运行日志

七、常见问题

报告显示昨天日期？GitHub Actions cron 可能延迟或跳过。去日志页确认 → 手动触发 → 等待约1分钟刷新
页面打开慢？GitHub Pages 服务器在海外，首次加载 2-5 秒属正常。可在管理面板触发后耐心等待
API 超时/数据缺？系统有自动重试机制（最多3次），失败后会记录到日志。缺数据时报告显示"暂缺"而非报错
持仓数据丢失？持仓保存在浏览器 localStorage。清除浏览器数据会丢失，建议偶尔手动备份
